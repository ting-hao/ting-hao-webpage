Robot dynamics

Consider the torque \(\mathbf{\tau}\) is generated by motor dynamics, the robot dynamics can be addressed based on the command signal \(\mathbf{u}\) as:

\[\begin{array}{r} B\left( \mathbf{q} \right)\ddot{\mathbf{q}} + C\left( \mathbf{q,}\dot{\mathbf{q}} \right) + F\dot{\mathbf{q}} + g\left( \mathbf{q} \right) = \mathbf{u}\#\left( \mathbf{}\ SEQ\ 方程式\ \backslash*\ ARABIC\ \mathbf{}1\mathbf{} \right) \end{array}\]

\[F = F_{v} + K_{r}K_{t}R_{a}^{- 1}K_{v}K_{r},\ \mathbf{u} = K_{r}K_{t}R_{a}^{- 1}G_{v}\mathbf{v}_{c}\]

Where \(B\left( \mathbf{q} \right)\) is inertia matrix, \(C\left( \mathbf{q,}\dot{\mathbf{q}} \right)\) is Coriolis-centripetal matrix, \(g\left( \mathbf{q} \right)\) is gravitational, \(F_{v}\) is viscous friction, \(K_{r}\) is gear ratio matrix, \(K_{t}\) is motor torque constant matrix, \(R_{a}\) is motor armature resistance matrix, \(\mathbf{v}_{c}\) is motor armature voltage, \(K_{v}\) is motor voltage constant matrix, \(G_{v}\) is amplifier gain matrix.

In this section, we’re going to use Matlab to simulate some of the controllers on IRISS v2 robot used for robotic eye surgery.

Joint Space Control

PD control with gravity compensation

\[\begin{array}{r} \mathbf{u} = g\left( \mathbf{q} \right) + K_{p}\widetilde{\mathbf{q}} - K_{d}\dot{\mathbf{q}}\mathbf{\#}(\ SEQ\ 方程式\ \backslash*\ ARABIC\ 2) \end{array}\]

\[\widetilde{\mathbf{q}} = \mathbf{q}_{d} - \mathbf{q}\]

Where \(\mathbf{q}_{d}\) is desired joint position. Since there is only joint position command in this controller, it’s usually used for pose maintenance.

Inverse dynamics control (Computed torque control)

Based on the robot dynamics model and the double integrator relationship, inverse dynamics control can achieve trajectory following with desired joint position, velocity and acceleration command.

\[\begin{array}{r} \mathbf{u} = B\left( \mathbf{q} \right)\mathbf{y} + n\left( \mathbf{q,}\dot{\mathbf{q}} \right)\mathbf{\#}(\ SEQ\ 方程式\ \backslash*\ ARABIC\ 3) \end{array}\]

\[n\left( \mathbf{q,}\dot{\mathbf{q}} \right) = C\left( \mathbf{q,}\dot{\mathbf{q}} \right) + F\dot{\mathbf{q}} + g\left( \mathbf{q} \right)\]

\[\begin{array}{r} \ddot{\mathbf{q}} = \mathbf{y} = {\ddot{\mathbf{q}}}_{d} + K_{d}\dot{\widetilde{\mathbf{q}}} + K_{p}\widetilde{\mathbf{q}}\#(\ SEQ\ 方程式\ \backslash*\ ARABIC\ 4) \end{array}\]

\[\dot{\widetilde{\mathbf{q}}} = {\dot{\mathbf{q}}}_{d} - \dot{\mathbf{q}},\ \widetilde{\mathbf{q}} = \mathbf{q}_{d} - \mathbf{q}\]

Where \(\mathbf{q}_{d}\) is desired joint position, \({\dot{\mathbf{q}}}_{d}\) is desired joint velocity, and \({\ddot{\mathbf{q}}}_{d}\) is desired joint acceleration.

Although inverse dynamics control works perfectly while the estimated model matches the actual robot dynamics, the tracking performance decreases when they’re not the same.

Robust control

Robust control ensures the asymptotical stability of the controlled system while the model uncertainty exists.

\[\begin{array}{r} \mathbf{u} = \widehat{B}\left( \mathbf{q} \right)\mathbf{y} + \widehat{n}\left( \mathbf{q,}\dot{\mathbf{q}} \right)\mathbf{\#}(\ SEQ\ 方程式\ \backslash*\ ARABIC\ 5) \end{array}\]

\[\begin{array}{r} \mathbf{y} = {\ddot{\mathbf{q}}}_{d} + K_{d}\dot{\widetilde{\mathbf{q}}} + K_{p}\widetilde{\mathbf{q}} + \rho\frac{\mathbf{z}}{\left\| \mathbf{z} \right\|}\mathbf{\#}(\ SEQ\ 方程式\ \backslash*\ ARABIC\ 6) \end{array}\]

\[\mathbf{z} = D^{T}Q\mathbf{\xi}\]

\[D = \begin{bmatrix} 0 \\ I \end{bmatrix}\mathbf{,\ \xi} = \begin{bmatrix} \widetilde{\mathbf{q}} \\ \dot{\widetilde{\mathbf{q}}} \end{bmatrix}\]

Where the estimated matrices, \(\widehat{B}\left( \mathbf{q} \right)\) and \(\widehat{n}\left( \mathbf{q,}\dot{\mathbf{q}} \right)\), are different from the actual ones, \(B\left( \mathbf{q} \right)\) and \(n\left( \mathbf{q,}\dot{\mathbf{q}} \right)\). And, \(Q\) is a positive-definite matrix, both \(\rho\) and \(Q\) are tuning parameters depending on the size of model uncertainty.

Adaptive control

To draw the issue of model uncertainty, the other method is adaptive control, which learns model parameters iteratively online based on the robot dynamic response to the input motor command.

The dynamic model is reformulated into:

\[\begin{array}{r} Y\left( \mathbf{q},\dot{\mathbf{q}},\ddot{\mathbf{q}} \right)\mathbf{\pi} = \mathbf{u}\#(\ SEQ\ 方程式\ \backslash*\ ARABIC\ 7) \end{array}\]

Where \(Y\left( \mathbf{q},\dot{\mathbf{q}},\ddot{\mathbf{q}} \right)\) is regressor matrix, and \(\mathbf{\pi}\) is a vector of constant dynamic model parameters such as link length and offset.

The control law is:

\[\begin{array}{r} \mathbf{u} = Y\left( \mathbf{q},\dot{\mathbf{q}},{\dot{\mathbf{q}}}_{r},{\ddot{\mathbf{q}}}_{r} \right)\widehat{\mathbf{\pi}} + K_{d}\mathbf{\sigma\#}(\ SEQ\ 方程式\ \backslash*\ ARABIC\ 8) \end{array}\]

\[{\dot{\mathbf{q}}}_{r} = {\dot{\mathbf{q}}}_{d} + \Lambda\widetilde{\mathbf{q}}\]

\[{\ddot{\mathbf{q}}}_{r} = {\ddot{\mathbf{q}}}_{d} + \Lambda\dot{\widetilde{\mathbf{q}}}\]

\[\mathbf{\sigma} = \dot{\widetilde{\mathbf{q}}} + \Lambda\widetilde{\mathbf{q}}\]

Where \(\widehat{\mathbf{\pi}}\) is estimated model parameter vector, and \(\Lambda\) is a positive-definite (usually diagonal) matrix.

The adaptive law is:

\[\begin{array}{r} \dot{\widehat{\mathbf{\pi}}} = K_{\pi}^{- 1}Y^{T}\left( \mathbf{q},\dot{\mathbf{q}},{\dot{\mathbf{q}}}_{r},{\ddot{\mathbf{q}}}_{r} \right)\mathbf{\sigma}\#(\ SEQ\ 方程式\ \backslash*\ ARABIC\ 9) \end{array}\]

Where \(K_{\pi}\) is a positive-definite matrix that determines the rate of convergence of dynamic model parameters to their asymptotic values.

Operational Space Control

Analytical Jacobian \(J_{A}(\mathbf{q})\) is a transformed version of geometric Jacobian \(J(\mathbf{q})\) which is obtained by differentiating forward kinematic equations. Analytical Jacobian is necessary for operational space control, which requires to represent end-effector orientation and changes in end-effector orientation. Analytical Jacobian maps joint angle velocity into angular velocity expressed in rates of change in Euler angle. Geometric Jacobian maps joint angle velocity into angular velocity vector \(\mathbf{\omega}\) which is more physical meaningful.

\[\begin{array}{r} \dot{\mathbf{x}} = \begin{bmatrix} \dot{\mathbf{p}} \\ \dot{\mathbf{\phi}} \end{bmatrix} = J_{A}\left( \mathbf{q} \right)\dot{\mathbf{q}}\#(\ SEQ\ 方程式\ \backslash*\ ARABIC\ 10) \end{array}\]

\[\begin{array}{r} \mathbf{v} = \begin{bmatrix} \dot{\mathbf{p}} \\ \mathbf{\omega} \end{bmatrix} = J\left( \mathbf{q} \right)\dot{\mathbf{q}} = \begin{bmatrix} I & 0 \\ 0 & T\left( \mathbf{\phi} \right) \end{bmatrix}\begin{bmatrix} \dot{\mathbf{p}} \\ \dot{\mathbf{\phi}} \end{bmatrix} = T_{A}\left( \mathbf{\phi} \right)J_{A}\left( \mathbf{q} \right)\dot{\mathbf{q}}\mathbf{\#}(\ SEQ\ 方程式\ \backslash*\ ARABIC\ 11) \end{array}\]

Where \(\mathbf{\phi}\) is 3D orientation using minimal representation (ie. Euler angle), and \(T(\mathbf{\phi})\) is a rotational matrix that uses the parameters from minimal representation of 3D orientation.

PD control with gravity compensation

\[\begin{array}{r} \mathbf{u} = g\left( \mathbf{q} \right) + J_{A}^{T}\left( \mathbf{q} \right)K_{p}\widetilde{\mathbf{x}} - J_{A}^{T}\left( \mathbf{q} \right)K_{d}J_{A}\left( \mathbf{q} \right)\dot{\mathbf{q}}\mathbf{\#}(\ SEQ\ 方程式\ \backslash*\ ARABIC\ 12) \end{array}\]

\[\widetilde{\mathbf{x}} = \mathbf{x}_{d} - \mathbf{x}\]

Where \(\mathbf{x}_{d}\) is desired position and orientation in operational space.

Inverse dynamics control

\[\begin{array}{r} \mathbf{u} = B\left( \mathbf{q} \right)\mathbf{y} + n\left( \mathbf{q,}\dot{\mathbf{q}} \right)\mathbf{\#}(\ SEQ\ 方程式\ \backslash*\ ARABIC\ 13) \end{array}\]

\[\begin{array}{r} \ddot{\mathbf{q}} = \mathbf{y} = J_{A}^{- 1}\left( \mathbf{q} \right)\left( {\ddot{\mathbf{x}}}_{d} + K_{d}\dot{\widetilde{\mathbf{x}}} + K_{p}\widetilde{\mathbf{x}} - {\dot{J}}_{A}\left( \mathbf{q,}\dot{\mathbf{q}} \right)\dot{\mathbf{q}} \right)\#(\ SEQ\ 方程式\ \backslash*\ ARABIC\ 14) \end{array}\]

\[\dot{\widetilde{\mathbf{x}}} = {\dot{\mathbf{x}}}_{d} - \dot{\mathbf{x}},\ \widetilde{\mathbf{x}} = \mathbf{x}_{d} - \mathbf{x}\]

Where \(\mathbf{x}_{d}\) is desired position and orientation, and \({\dot{\mathbf{x}}}_{d}\)/\({\ddot{\mathbf{x}}}_{d}\) is desired velocity/acceleration and angular velocity/acceleration in operational space.

Force Control

Now, we start to consider the force exerted from the robot to the environment \(\mathbf{h}\), and the robot dynamics can be written as:

\[\begin{array}{r} B\left( \mathbf{q} \right)\ddot{\mathbf{q}} + C\left( \mathbf{q,}\dot{\mathbf{q}} \right) + F\dot{\mathbf{q}} + g\left( \mathbf{q} \right) = \mathbf{u} - J^{T}\left( \mathbf{q} \right)\mathbf{h}\#(\ SEQ\ 方程式\ \backslash*\ ARABIC\ 15) \end{array}\]

For the previous controllers, no matter how much the force exerted on the robot, the robot will keep moving to the exact desired location. In some circumstance, this is quite dangerous, which can hurt people or damage the environment. Therefore, we want the robot can interact with the environment, and response to the external forces.

Impedance control

The error vector \(\widetilde{\mathbf{x}} = \mathbf{x}_{d} - \mathbf{x}\) in previous section will be expressed in the desired orientation frame:

\[\begin{array}{r} \widetilde{\mathbf{x}} = \begin{bmatrix} R_{d}^{T}\left( \mathbf{p}_{d} - \mathbf{p} \right) \\ \mathbf{\phi}_{d} - \mathbf{\phi} \end{bmatrix}\#(\ SEQ\ 方程式\ \backslash*\ ARABIC\ 16) \end{array}\]

And, the PD controller becomes:

\[\begin{array}{r} \mathbf{u} = g\left( \mathbf{q} \right) + J_{A_{d}}^{T}\left( \mathbf{q},\widetilde{\mathbf{x}} \right)\left( K_{p}\widetilde{\mathbf{x}} - K_{d}J_{A_{d}}\left( \mathbf{q},\widetilde{\mathbf{x}} \right)\dot{\mathbf{q}} \right)\mathbf{\#}(\ SEQ\ 方程式\ \backslash*\ ARABIC\ 17) \end{array}\]

\[\begin{array}{r} J_{A_{d}}\left( \mathbf{q},\widetilde{\mathbf{x}} \right) = \begin{bmatrix} I & 0 \\ 0 & T^{- 1}\left( \mathbf{\phi} - \mathbf{\phi}_{d} \right) \end{bmatrix}\begin{bmatrix} R_{d}^{T} & 0 \\ 0 & R_{d}^{T} \end{bmatrix}J\left( \mathbf{q} \right)\#(\ SEQ\ 方程式\ \backslash*\ ARABIC\ 18) \end{array}\]

In addition, inverse dynamics control becomes:

\[\begin{array}{r} \mathbf{u} = B\left( \mathbf{q} \right)\mathbf{y} + n\left( \mathbf{q,}\dot{\mathbf{q}} \right) + J^{T}\left( \mathbf{q} \right)\mathbf{h\#}(\ SEQ\ 方程式\ \backslash*\ ARABIC\ 19) \end{array}\]

\[\begin{array}{r} \mathbf{y} = J_{A_{d}}^{- 1}\left( \mathbf{q},\widetilde{\mathbf{x}} \right)M_{d}^{- 1}\left( K_{d}\dot{\widetilde{\mathbf{x}}} + K_{p}\widetilde{\mathbf{x}} - M_{d}{\dot{J}}_{A_{d}}\left( \mathbf{q},\widetilde{\mathbf{x}},\dot{\mathbf{q}},\dot{\widetilde{\mathbf{x}}} \right)\dot{\mathbf{q}}\mathbf{+}M_{d}\dot{\mathbf{b}} - \mathbf{h}^{d} \right)\mathbf{\#}(\ SEQ\ 方程式\ \backslash*\ ARABIC\ 20) \end{array}\]

Where \(M_{d}\), \(K_{d}\) and \(K_{p}\) are tunable matrices, and \(\mathbf{h}^{d}\) is the force represented on the desired pose frame.

After plugging \((19)\) into \((15)\), the dynamic can be simplified to linear impedance relationship independent of the configuration of the manipulator:

\[M_{d}\ddot{\widetilde{\mathbf{x}}} + K_{d}\dot{\widetilde{\mathbf{x}}} + K_{p}\widetilde{\mathbf{x}} = \mathbf{h}^{d}\]

When the end-effector exerts a constant force along positive z-axis, which means there might be an obstacle near the desired position, its position has never reached the desired value. In other words, the end-effector gently touched the obstacle instead of colliding into that violently to reach the target.
